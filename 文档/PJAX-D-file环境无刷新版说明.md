# PJAX D 方案：file:// 环境无刷新版

## 这版解决的问题

上一版 D 方案依赖 `fetch(目标HTML)` 来读取目标页面，再只替换右侧 `.main` 正文。

但你当前是在 `file:///G:/...` 环境中打开文档。很多浏览器/Chromium WebView 会禁止页面用 `fetch` 读取本地 HTML 文件，于是 PJAX 失败，自动回退成普通跳转，导致每次点击都会重新触发 `pageshow`，页面仍然闪动。

这版新增了：

```text
_资源/页面内容索引.js
```

它把所有 HTML 页面右侧 `.main` 正文预先打包成一个 JS 索引。浏览器允许 `file://` 页面加载本地 JS，所以点击目录时不再 `fetch` HTML，而是直接从内存索引里取正文并替换。

## 应该看到的控制台日志

点击站内目录时，正常应该看到：

```text
[文档切页调试] 已拦截点击，准备局部切页
[文档切页调试] 开始局部切页
[文档切页调试] file:// 环境准备加载页面内容索引，用它替代 fetch 读取 HTML
[文档切页调试] 页面内容索引加载完成
[文档切页调试] 命中页面内容索引，不使用 fetch，因此不会触发 file:// 安全限制
[文档切页调试] 开始替换右侧正文 .main，左侧目录不会重建
[文档切页调试] 局部切页完成
```

正常 PJAX 点击时，不应该每次都重新出现：

```text
[文档切页调试] 页面生命周期 pageshow
```

如果它仍然每次都出现，说明点击没有被脚本接管，或者目标链接没有被判断为站内 HTML 链接。

## 稳定后关闭日志

打开：

```text
_资源/文档脚本.js
```

把：

```js
const PJAX_DEBUG = true;
```

改成：

```js
const PJAX_DEBUG = false;
```
