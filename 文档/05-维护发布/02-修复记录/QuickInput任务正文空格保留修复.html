QuickInput 任务正文空格保留修复

问题

示例任务：

- [x] 🖥 长治学院  设计道旗定稿 (主题::工作/设计) (时间::09:00) (结束::09:30) (时长::30) (模板ID::ovr_mfhzw9d975by6) (模板来源::override)   ✅ 2026-04-09

期望正文是：

长治学院  设计道旗定稿

这里 长治学院 和 设计道旗定稿 中间有两个空格，属于用户输入内容。旧的正文清理逻辑最后使用 /\s+/g 把所有连续空白压成一个空格，导致编辑回填时看起来像正文没有被完整读取。

修改

修改文件：

  • src/core/utils/text.ts

stripTaskLineToEditableText() 仍然会移除：

  • 任务 checkbox 前缀
  • 行首图标 / 优先级装饰
  • (key::value) 元数据
  • 📅 / ⏳ / 🛫 / ➕ / ✅ / ❌ 日期类元数据
  • #tag
  • recurrence 信息

但最后不再把正文内部的连续空格整体压缩，只清理换行、tab 和首尾空白。

验收标准

输入：

- [x] 🖥 长治学院  设计道旗定稿 (主题::工作/设计) (时间::09:00) (结束::09:30) (时长::30) (模板ID::ovr_mfhzw9d975by6) (模板来源::override)   ✅ 2026-04-09

编辑正文应为：

长治学院  设计道旗定稿

不能变成：

长治学院

也不应被压成：

长治学院 设计道旗定稿
