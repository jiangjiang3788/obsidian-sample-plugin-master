shared/ 总体

src/shared/
  • 像什么像店里的公共工具柜。
  • 一句话这里放的是多个区域都会来借用的公共工具和零件，不属于某一个单独服务区。
  • 它在哪层它不是前台服务区，也不是后仓核心区，而是横向支撑层。
  • 主要干什么减少重复，把多个地方都要用的组件、hooks、通用小工具集中起来。
  • 和 core 的区别core 是基础能力区，负责撑住系统；shared 是公共复用区，负责减少重复。
  • 你怎么记看到 shared，就想成“大家都能来借，但不是谁的私人物品”。
你以后整理时最关键的一问是：这个东西是不是至少两个区域都会长期用？ 如果不是，就要小心它不该进 shared。

components/

src/shared/components/
  • 像什么像公共零件盒。
  • 一句话这里放的是多个地方都能复用的界面小零件，不只服务一个功能区。
  • 它在哪层属于 shared 里最常见的一层，因为很多复用都会先体现在 UI 组件上。
  • 主要干什么提供通用按钮、选择器、列表块、树形选择等能在多个功能区重复使用的界面零件。
  • 和 features 里的组件区别shared/components 是多个服务区共用的，features 里的组件通常只服务某一个具体功能区。
  • 你怎么记看到 shared/components，就想成“公共零件盒，谁需要就拿”。
如果一个组件看起来很通用，但实际上只有 settings 自己在用，那它更应该留在 settings，而不是提前放进 shared。

hooks/

src/shared/hooks/
  • 像什么像常用操作小技巧卡。
  • 一句话把多个地方都会重复写的界面逻辑，整理成可重复使用的小套路。
  • 它在哪层属于 shared 里的逻辑复用层，比 components 更偏行为，不只是长相。
  • 主要干什么封装状态读取、事件处理、界面交互、重复性的前端逻辑。
  • 和 app/store 的区别hooks 更像“拿来就用的小套路”，app/store 是系统真正的状态中心。
  • 你怎么记看到 shared/hooks，就想成“多个店员都会抄的小技巧卡片”。
如果一个 hook 强烈依赖某个 feature 的业务语义，那就别急着放 shared，它可能属于那个 feature 自己。

utils/

src/shared/utils/
  • 像什么像公共小工具抽屉。
  • 一句话这里放的是很多地方都可能顺手要用到的小工具，不属于某个单独功能区。
  • 它在哪层属于 shared 里的轻量辅助层，偏短小、通用、可随手复用。
  • 主要干什么做轻量转换、拼接、辅助判断、简单格式化这类公共小动作。
  • 和 core/utils 的区别core/utils 更偏系统底层常用工具，shared/utils 更偏前台多个区域都会顺手用的公共小工具。
  • 你怎么记看到 shared/utils，就想成“公共抽屉里的剪刀、胶带、小尺子”。
如果一个工具已经明显带着系统底层语义，就别放 shared/utils，可能更适合放 core/utils。

ThemeTreeSelect 这类内容

shared/components/...（示意）
  • 像什么像公共展示支架。
  • 一句话它看起来有点业务味，但如果多个地方都会复用，它仍然可以放在 shared。
  • 它在哪层属于 shared 里比较“专用但可复用”的那类公共件。
  • 主要干什么给多个区域提供相同的树形选择、展示、交互支架，而不是只给一个单独页面用。
  • 最容易误判的点很多人看到“主题树”就以为只能属于 settings，但如果别的地方也要选树，它就可以是公共件。
  • 你怎么记看到这种内容，就问自己：它是“某个柜台专属”，还是“多个柜台都会摆的公共支架”？
这类内容正好说明：shared 不是完全没有业务味，而是要看它是不是“跨多个区域复用”。

shared 的放置规则

整理时最重要的一页
  • 能放进 shared 的至少两个区域都会长期用，而且不强依赖某个单独 feature 的内容。
  • 不要放进 shared 的暂时不知道放哪的代码、只服务一个功能区的私有组件、带很强业务名词的专属逻辑。
  • 整理时先问这是“公共工具”，还是“某个服务区的私人物品”？
  • 最常见错误为了看起来整洁，把大量 feature 私有内容提前抽到 shared，结果 shared 反而最乱。
  • 你该怎么做先保守一点：宁可先留在 feature 内，等确认真的复用多次，再抽进 shared。
  • 一句话总原则shared 解决“重复使用”，不是解决“我现在懒得想该放哪”。
如果你现在准备整理文件夹，我建议你把 shared 当成“最后再抽公共件”的地方，而不是一开始就往里塞。
