QuickInput 任务内容提取调试说明

为什么要重新审视任务内容提取

示例任务：

- [x] 🖥 长治学院  设计道旗定稿 (主题::工作/设计) (时间::09:00) (结束::09:30) (时长::30) (模板ID::ovr_mfhzw9d975by6) (模板来源::override)   ✅ 2026-04-09

真正的内容应该是：

长治学院  设计道旗定稿

这次重新审视后，结论是：任务内容不能再从 title 取，也不能按空格 split。正确方式应该是从原始任务行中剥离任务语法、开头图标、元数据、tag 后，保留中间整段正文。

本次修改

  1. stripTaskLineToEditableText() 只剥离任务语法和元数据，不压缩正文内部连续空格。
  2. parser 同时写入 extra[正文] 和 extra[内容]，兼容旧模板字段。
  3. 编辑回填识别范围扩大到 内容 / 正文 / content / body / 任务内容 / 记录内容 / semanticType=content。
  4. 增加中文调试输出。

怎么打开调试

打开开发者控制台执行：

window.__THINK_RECORD_DEBUG__ = true

然后重新打开编辑任务。重点看：

  • [记录调试][任务读取] parser.parseTaskLine 正文提取
  • [记录调试][编辑读取]任务内容字段回填决策
  • [记录调试][编辑读取]任务非内容字段读取检查

怎么判断问题在哪

  • 如果 fromRaw 是完整的，但界面不完整：说明后续表单初始化、主题默认值或字段映射又覆盖了内容。
  • 如果 fromRaw 本身不完整：说明正文剥离规则还要继续补。
  • 如果 是否识别为任务内容字段 = false：说明模板字段名没有被识别为正文，需要改字段 key/label 或加 semanticType: content。
