模块职责

定义模块负责什么、不负责什么，防止代码职责漂移。

状态：建议模板 最后更新：2026-05-07 格式：HTML

模块职责表

模块负责不负责
main.ts生命周期、命令注册、设置加载、组合入口复杂业务、UI 细节、数据算法。
app服务组合、runtime、usecases、capabilities、feature 注册直接操作 Obsidian 细节。
core业务服务、数据结构、算法、类型、工具直接依赖插件 UI 或 Obsidian DOM。
platform把 Obsidian API 包装成 port 实现业务规则和 UI 组件状态。
features具体用户功能、UI 容器、命令注册跨项目的基础工具和核心算法。
shared可复用 UI、hook、工具和样式某个功能的专属业务逻辑。

职责判断问题

  • 这个代码能不能离开 Obsidian 环境测试？如果能，更适合 core。
  • 这个代码是不是只适合某个功能？如果是，更适合 features。
  • 这个代码是不是 Obsidian API 包装？如果是，放 platform。
  • 这个代码是不是多个功能复用？如果是，考虑 shared。
