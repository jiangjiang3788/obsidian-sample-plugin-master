## 表 2：专门针对“表格编辑 / 拖拽 + 写回”的测试计划（你最在意的那块）

### 2.1 单测层（最底层：读→改→写协议）

| 编号   | 范围            | 目的                                      | 用例描述（可以直接变成 Jest）                                                                                                                    | 关联文件                  |
| ---- | ------------- | --------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------ | --------------------- |
| UT-1 | 解析（read）      | 确保按 `start/end` 区间 + `[!thinktxt]` 解析稳定 | 输入：包含 `<!-- thinktxt:start --> ... <!-- thinktxt:end -->` 以及多条 `{key:: value}` 行的 markdown；<br>输出：数组 rows，字段完整（id/title/status/...）。 | parser.ts + DataStore |
| UT-2 | 更新（updateRow) | 改一个单元格不会破坏其它内容                          | 给定 mdBefore + 操作 `updateRow(id=2, patch={title: 'BBB-EDIT'})`；<br>输出 mdAfter：<br>1）只有目标行内容变；2）缩进/其它行保持不变；3）`start/end` 注释位置不变。       | patcher（你要写的模块）       |
| UT-3 | 插入/删除         | 保证 start/end 块范围仍正确                     | `insertRow(afterId=2, row=...)` + `deleteRow(id=1)`；<br>验证：新增/删除行都在 `start/end` 内，区块外内容完全不变。                                         | 同上                    |
| UT-4 | 拖拽排序（reorder） | 保证顺序正确、空行/注释不乱                          | 输入 3 行 + 若干空行/注释；操作：`reorder(0→2)`；<br>输出：3 行顺序调换，但所有空行/注释保持原位置（或者按你设计的明确规则）。                                                        | 同上                    |
| UT-5 | 多块/多区间边界      | 确保只改目标区间，不污染别的 block                    | 文档里有两个 `start/end` 区间；对第一个区间执行一组 ops；<br>验证第二个区间一点没动。                                                                                | 同上                    |

**原则**：

* 单测里只碰字符串，不碰 Obsidian、不碰 UI。
* 一条用例 = 一个“金样本 md 前后对照”，以后重构 patcher 都靠它。

### 2.3 端到端层（你最在意的部分）

| E2E-4 | 改设置（例如过滤条件） → 仪表盘/预览内容随之变化             | 1. 在 Settings 里修改 thinktxt 相关设置（比如只显示某 tag / 某路径）。<br>2. 回到仪表盘或预览。<br>3. 验证行数/内容发生对应变化。                                          | Phase2 / Settings；Phase0 P0 测试框架；SettingsRepo |

1. **先做 E2E-1（纯只读预览）**：这个实现成本最低，而且能证明 pipeline 通了。
2. 然后并行推进：

   * 单测层 UT-1~UT-4（patcher）
   * 集成 IT-1（DataStore+patcher）
3. patcher 相对稳定后，再补 **E2E-2 / E2E-3**。
4. 最后再加 E2E-5（生命周期 + reload），给整套东西兜底。
