# QuickInput 路径变化保存策略更新

## 背景

早期安全 MVP 对编辑时的目标路径变化采取“一律阻止保存”。这个策略能避免误迁移，但实际使用中过于保守：当用户确实切换了主题/模板并希望记录跟随新目标位置保存时，会被拦住。

## 当前策略

当前策略改为安全迁移保存：

1. 目标路径不变：原地更新。
2. 目标路径变化：先写入新位置，再删除旧记录。
3. 删除旧记录失败：保留新旧两条记录，并返回 partial_success，提示用户手动清理。

## 风险说明

这个策略比阻止保存更实用，但风险也更高。后续需要继续强化：

- 新记录定位。
- 旧记录删除冲突检测。
- 保存前 UI 提示。
- 手测验收清单。
