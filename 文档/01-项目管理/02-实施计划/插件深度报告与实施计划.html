S2 S2-3 极高 识别依据确认 明确对象识别到底靠什么 盘点来源事实、模板信息、能力开关各自角色 Task / Block 判定矩阵 否 Thinking S2-1/S2-2 完成 能回答为什么当前对象是 Task 或 Block 识别依据不再靠猜字段名或猜模板
S2 S2-4 极高 入口链路确认 明确每个编辑入口如何构造上下文 盘点列表、详情、搜索、时间轴、快速输入等入口 上下文入口总表 否 Thinking S2-2 完成 所有入口都能对应到上下文构造链 能指出哪些入口最容易串对象、丢能力、错字段

S3 S3-7 极高 提交前归一确认 明确保存在提交前会不会再次调整 盘点保存前归一化、二次修正、冲突处理 提交前归一表 否 Thinking S3-6 完成 能回答保存前是否会再推导一次 再打开记录时字段关系与编辑时保持一致
S3 S3-8 极高 写回规则确认 明确最终如何落盘、如何再次读取 盘点写回格式、读取重建、稳定性 写回链路表 否 Thinking S3-7 完成 能回答最终怎么写、怎么读 时间相关记录写回后再次读取仍符合既定规则
S3 S3-9 极高 时间函数全表确认 形成时间系统完整函数地图 盘点所有时间函数文件、函数名、输入、输出、调用方、风险点 时间函数总表 否 Thinking S3-1 至 S3-8 完成 所有时间函数被完整列出 不再存在“还有一处时间逻辑没看到”的盲区
S4 P0-1 极高 编辑上下文链改造 修“编辑错对象、读错上下文” 统一上下文构造、统一入口、统一能力判断 P0 改造清单（上下文） 是 Thinking 设计 / Pro 落地 S1/S2/S3 完成 能稳定识别当前编辑对象 所有主要入口均不再串任务和 block
S4 P0-2 极高 时间推导链改造 修“时间推导不符合使用习惯” 统一时间推导总入口、联动规则、默认方向、跨天与日期归属 P0 改造清单（时间） 是 Thinking 设计 / Pro 落地 S1/S2/S3 完成 时间联动符合既定规则 跨天、反推、再次编辑、再次读取全部稳定
S4 P0-3 高 调试能力补齐 让错误可观测、可复盘 增加上下文调试、时间推导调试、写回前快照 P0 调试清单 是 Pro P0-1/P0-2 启动 能看到关键链路调试信息 能复盘一次完整编辑到写回过程
S4 P1-1 高 策略对象沉淀 把 P0 规则从页面逻辑沉淀到策略层 提炼上下文策略、时间策略、能力策略 P1 策略清单 是 Thinking 设计 / Pro 落地 P0 完成 规则不再散落在 UI 和 usecase 后续加规则只需改策略层而非全仓补丁
S4 P1-2 高 统一入口 不同入口复用同一规则 统一列表、详情、搜索、时间轴、快速输入等入口逻辑 P1 入口统一清单 是 Pro P1-1 启动 修一处不再在别处复发 多入口编辑同一对象结果一致
S4 P1-3 高 统一写回 输入、保存、再次读取一致 统一提交前归一、写回格式、回读重建 P1 写回统一清单 是 Pro P1-1 启动 再打开记录时字段关系稳定 输入中联动、保存后结果、再次读取三者完全一致
S4 P2-1 中 完整字段系统 从核心字段契约扩展到完整字段协议 输入 / 展示 / 筛选 / 存储协议扩展 P2 字段系统清单 是 Thinking 设计 / Pro 落地 P1 完成 新增字段不再牵一发动全身 字段系统可支撑未来更多模板和命名映射
S4 P2-2 中 目录收口 提升可维护性和定位效率 收口时间、上下文、字段相关代码目录 P2 目录清单 是 Pro P2-1 启动或并行 关键代码可快速定位 目录与职责基本对齐且不引入回退
S4 P2-3 中低 媒体进一步落地 让图片 / 音频 / 附件进入真实流程 占位后的媒体字段逐步接入展示、编辑、写回 P2 媒体清单 是 Thinking 设计 / Pro 落地 P2-1 完成 媒体字段可进入真实流程 在既有契约下扩展，不破坏主线
