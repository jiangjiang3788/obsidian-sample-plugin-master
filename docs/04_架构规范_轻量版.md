## 分层（推荐）
- core：启动编排与生命周期  
- services：封装 Obsidian API（vault/events/metadata/timer…）  
- features：业务规则与功能注册（dashboard/settings/…）  
- ui：界面展示与输入（可选独立）  

---

## 8 条铁律（AI 必须遵守）
1) Markdown 是真相：cache 只能加速  
2) 写回必须原位：定位到原始行/区块再改  
3) Obsidian API 访问要收敛：尽量经 ports/services  
4) UI 不写业务：展示/输入；业务在 feature；读写在 service  
5) services 不依赖 UI  
6) feature 之间不互相 import 内部实现：跨模块只用 public 接口  
7) main/bootstrap 不写业务：只做注册编排  
8) 写回动作必须可验证：写回后重新解析确认一致；失败要可恢复
