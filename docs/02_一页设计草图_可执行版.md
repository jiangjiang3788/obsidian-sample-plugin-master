# ThinkOS 插件：一页设计草图（可执行版）
日期：2026-02-28

> 用途：把“事实复盘”抽象成：目标、流程、数据、验收、边界。你已确认 ✅

---

## 1）一句话（Why）
在 Obsidian 中实现：快速记录 → 落盘为 Markdown → 扫描为 items（任务 + KV 块）→ Dashboard 展示/筛选 →（部分）写回 Markdown 原位。

---

## 2）本阶段 MVP（必须交付）
1) Dashboard 稳定渲染：` ```think <LayoutName> ``` ` 能显示对应布局  
2) 扫描稳定产出 items：任务行 + KV 块（callout/thinktxt）为第一优先级  
3) 写回策略落地（部分）：至少 2 个写回动作（勾选完成、改时间），且写回原位  
4) 自助恢复能力：重建索引/刷新数据等命令可用  

---

## 3）用户流程
1) 用户写入任务行或 KV 块  
2) 插件扫描/监听变化 → 更新 items  
3) Dashboard 按 layout 展示  
4) 用户在 Dashboard 操作（部分）→ 写回原位 → 重新扫描 → 展示更新  

---

## 4）数据模型（人话版）
- Item：来源 taskLine / kvBlock（future: frontmatter），包含 filePath + position + fields + status  
- Layout：Dashboard 布局（settings）  
- Settings：插件设置（layouts、输入模板等）  
- Cache：扫描缓存（仅加速）  

---

## 5）验收标准（最小）
- 代码块渲染稳定  
- 重启后可从 Markdown 恢复同样展示  
- 勾选完成、改时间：都能写回原位，并且重新扫描后展示一致
