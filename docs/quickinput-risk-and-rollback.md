# QuickInput 风险与回滚说明

## 高风险点

### 1. 路径变化迁移保存

当前策略是：

1. 先写入新位置
2. 再删除旧记录
3. 删除失败则返回 partial_success

风险：

- 删除旧记录失败会产生一新一旧两条记录
- 这是有意保守设计，优先避免数据丢失

回滚方式：

- 将 `move_and_replace` 行为改回阻止保存
- 保留 OutputPlan 预览和一致性校验

### 2. 任务写回保真

风险：

- 旧任务行里有非常规格式时，保留逻辑可能追加不完全
- 新模板如果也输出同类字段，需要防重复

回滚方式：

- 保留原始任务行备份
- 如果发现重复字段，优先修去重规则，不要回退到整行盲替换

### 3. 时间联动

风险：

- 默认值注入和用户手动修改的优先级可能冲突

回滚方式：

- 只在初始化默认值阶段触发联动
- 用户手动修改阶段保留现有交互计算

### 4. Legacy fallback

风险：

- 旧 `editStateResolver` 仍存在 fallback
- 如果 snapshot 信息缺失，可能退回旧推断逻辑

处理原则：

- fallback 只允许兜底，不允许新增业务依赖
- 相关代码必须保留 `LEGACY-FALLBACK` 或 `CLOSEOUT-GUARD` 注释

## 推荐回滚顺序

如果线上出现严重问题，按这个顺序回滚：

1. 关闭迁移保存，恢复为路径变化阻止
2. 保留任务读取修复
3. 保留模板识别 guard
4. 保留保存位置预览，但不执行 move_and_replace
5. 最后再考虑回退时间联动

不要优先回退任务内容读取，因为它修复的是核心问题。
