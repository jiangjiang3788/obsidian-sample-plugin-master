## 3. 技术架构

### 3.1 分层结构

```
src/
├── main.ts              # 插件入口（生命周期、DI 容器、命令注册）
├── platform/            # Obsidian API 适配层
│   ├── Vault 端口实现
│   ├── UI 端口实现
│   ├── Modal 端口实现
│   ├── Events 端口实现
│   └── Metadata 端口实现
├── core/                # 核心领域能力
│   ├── settings/        # 设置管理
│   ├── storage/         # 存储抽象
│   ├── ports/           # 端口定义
│   └── utils/           # 核心工具（parser 等）
├── features/            # 功能模块
│   ├── timer/
│   ├── quickinput/
│   ├── aiinput/
│   ├── aichat/
│   └── settings/
├── shared/              # 跨模块共享
│   ├── public/          # 公共 API
│   └── private/         # 内部工具
└── app/                 # 应用装配与运行时
    ├── ServiceManager   # 服务生命周期管理
    ├── usecases/        # 用例编排
    ├── capabilities/    # 能力注册
    └── FeatureLoader    # 功能加载器
```

### 3.2 核心设计模式

- **依赖注入（DI）**：使用 DI 容器管理服务依赖
- **端口适配器**：通过 `src/platform/` 隔离 Obsidian API
- **服务管理**：ServiceManager 统一管理服务生命周期
- **功能模块化**：每个功能独立在 `src/features/<name>/`
- **边界约束**：通过 gate 脚本强制架构边界

### 3.3 工程能力

#### 质量保障

- **Lint**：代码风格检查
- **Typecheck**：TypeScript 类型检查
- **Test**：Jest 单元测试框架
- **Gate**：架构边界门禁（多个 gate 脚本）

#### 常用命令

```bash
npm run check      # 质量门禁（lint + typecheck + gate）
npm test           # 运行测试
npm run build      # 构建产物
```

#### 架构门禁（Gate）

位于 `scripts/*gate*.mjs`，包括：

- `obsidian-leak:gate` - 防止平台 API 泄漏到业务层
- `core-obsidian:gate` - core 层不依赖 obsidian
- `core-public:gate` - core 层 public API 约束
- `shared-public:gate` - shared 层 public API 约束
- `iconaction:gate` - IconAction 组件使用规范
- `di-resolve:gate` - DI 容器使用约束
- `events-boundary:gate` - 事件边界约束
- `check-boundaries` - 深导入边界检查
