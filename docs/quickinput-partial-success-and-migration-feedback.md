# QuickInput 迁移保存反馈收口

## 本次修改目的

上一阶段已经允许路径变化时执行安全迁移保存：先写入新位置，再删除旧记录。
但如果旧记录删除失败，结果是 `partial_success`，此前 UI 会把它当普通失败处理，容易造成两个问题：

1. 用户以为没有保存成功，可能再次点击保存，制造重复记录。
2. 新位置已经存在记录，但旧位置也保留，提示语不够明确。

## 本次行为

- `success`：正常成功，显示绿色/普通成功提示，并关闭 QuickInput。
- `partial_success`：按“带警告的成功”处理，显示较长中文提示，并关闭 QuickInput。
- `error / validation_error / conflict`：仍按失败处理，保留原行为。

## 为什么 partial_success 要关闭面板

迁移保存的 partial_success 表示：

- 新记录已经写入目标位置；
- 旧记录删除失败；
- 为避免数据丢失，旧记录被保留。

如果继续停留在编辑面板，用户再次保存可能继续写入新位置，造成更多重复项。因此关闭面板更安全。

## 验收标准

1. 修改主题导致目标文件变化时，保存后提示：`已迁移保存：原路径 → 新路径`。
2. 如果删除旧记录失败，提示：`已写入新位置，但旧记录删除失败；请检查并手动清理原位置`。
3. partial_success 不再显示红色“保存修改失败”。
4. partial_success 后面板关闭，避免重复保存。
