## v13.6 这轮我多做了哪些步

### ✅ Step A（Phase1 PR4）：TimelineRange 单一来源（类型 + 纯函数）

目标：把“年/季/月/周/天”的范围计算收敛到**一个地方**，后续替换调用点就不会各自算各自的。

新增/修改点：

1. **新增类型模型**

* `src/types/timeline/TimelineRange.ts`

  * `TimelineView = '年' | '季' | '月' | '周' | '天'`
  * `TimelineRange { view, anchor, start, end, key }`

2. **新增纯函数单一来源（ISO 周语义）**

* `src/core/utils/timelineRange.ts`

  * `normalizeTimelineView(view: string): TimelineView`（把遗留的 `'日'` 统一归一成 `'天'`，避免各处 any/cast）
  * `rangeKeyForView(anchor, view)`（比如 `2026-W03`）
  * `calculateTimelineRange(anchor, view)`（start/end 统一：周用 `isoWeek`）
  * `toIsoDateTuple(...)`（返回 `['YYYY-MM-DD','YYYY-MM-DD']`，便于筛选）

3. **对外暴露**

* `src/core/utils/index.ts` 已加入 `export * from './timelineRange'`
* 所以外部可以直接 `import { calculateTimelineRange } from '@core/public'`

### ✅ Step B（替换一个真实调用点）：LayoutRenderer 日期范围统一走 TimelineRange

目标：从“范围来源分散”迈出第一步：**最关键的布局范围**先统一。

* `src/features/settings/LayoutRenderer.tsx`
