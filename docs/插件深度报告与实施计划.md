## 固定节奏表

| 批次 | 阶段范围 | 目标 | 建议模型 | 结束标志 |
|---|---|---|---|---|
| 第一批 | S0 + S1 | 先完成深度报告与核心字段契约确认 | Thinking | 已形成深度报告总稿 + 核心字段契约表 |
| 第二批 | S2 + S3 | 再完成上下文模型与时间领域规则确认 | Thinking | 已形成上下文总表、时间函数总表、写回链路表、决策表 |
| 第三批 | S4 P0 | 按已确认规则开始止痛改造 | Thinking 设计 / Pro 落地 | 上下文链、时间链、调试链可稳定工作 |
| 第四批 | S4 P1 | 把 P0 的修复沉淀成稳定系统 | Thinking 设计 / Pro 落地 | 多入口一致、写回一致、规则不再散落 |
| 第五批 | S4 P2 | 进入完整重构与扩展 | Thinking 设计 / Pro 落地 | 字段系统、目录、媒体扩展进入可持续阶段 |

## 一句话总纲

> 先做深度报告，再确认核心字段契约，再确认上下文模型，再确认时间领域规则，最后才按 P0 / P1 / P2 节奏改代码；所有实施都必须服从前面确认出的真源。
