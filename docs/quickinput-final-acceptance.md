# QuickInput 最终验收清单

## 目标

本清单用于确认 QuickInput 本轮主线修改已经收口。重点不是继续扩功能，而是确认：任务读得准、时间算得准、写回不误伤、迁移保存可追踪。

## 必测用例

### 1. 任务内容读取

原始任务：

```md
- [x] 🖥 长治学院  设计道旗定稿 (主题::工作/设计) (时间::09:00) (结束::09:30) (时长::30) (模板ID::ovr_mfhzw9d975by6) (模板来源::override) ✅ 2026-04-09
```

验收：

- 编辑面板内容字段应显示：`长治学院  设计道旗定稿`
- 双空格应保留
- 不应只显示 `长治学院`
- 不应被识别成“闪念/block”模板

### 2. 任务模板识别

验收：

- 带 `(模板ID::xxx) (模板来源::override)` 的任务，优先使用原模板信息
- task 编辑时不应被 block/闪念模板抢走
- 保存目标不应无故跳到闪念文件

### 3. 时间计算

验收：

- `时间 + 时长` 能计算 `结束`
- `结束 + 时长` 能计算 `时间`
- `时间 + 结束` 能计算 `时长`
- 默认值注入后也应触发一次计算
- 用户手动修改后的字段不应被默认值无故覆盖

### 4. 主题字段

验收：

主题 `工作/设计/道旗` 应拆出：

- `rootTheme`: `工作`
- `leafTheme`: `道旗`
- `themePath`: `工作/设计/道旗`

视图设置中应能使用这三个字段筛选或展示。

### 5. 写回保真

验收：

保存后应保留：

- checkbox 状态，例如 `- [x]`
- 原有 tag
- `📅 / ⏳ / 🔁 / ✅` 等任务符号上下文
- `(key::value)` 元数据
- `(模板ID::xxx)` 和 `(模板来源::xxx)`

如果新模板已经输出同类字段，不应重复追加旧字段。

### 6. 路径变化迁移保存

验收：

- 路径不变：原地更新
- 路径变化：先写入新位置，再删除旧记录
- 删除旧记录失败：返回 partial_success，提示手动清理旧记录
- partial_success 后面板应关闭，避免重复点保存

### 7. 保存位置一致性

验收：

- UI 实时预览显示的目标位置应与实际提交重新计算的位置一致
- 如果不一致，应取消保存
- 取消保存时不得写入新记录，也不得删除旧记录

## 调试方式

控制台启用：

```js
window.__THINK_RECORD_DEBUG__ = true
```

重点看：

- 任务正文提取日志
- 模板解析日志
- 时间计算日志
- 保存位置 / 迁移保存日志
