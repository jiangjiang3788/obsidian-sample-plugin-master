# QuickInput 本轮修改总览

## 主线目标

本轮修改围绕 QuickInput 的任务编辑主链路收口：

1. 任务内容完整读取
2. 时间字段默认值与联动计算
3. 主题字段三分法：根主题 / 叶主题 / 完整路径
4. 任务写回保真
5. 保存位置实时预览
6. 路径变化迁移保存
7. partial_success 反馈收口
8. legacy fallback 降级，减少长期双轨

## 已完成

### 任务读取

- 增加统一任务正文提取入口
- 不再按空格切分任务正文
- 保留正文内部连续空格
- parser / edit resolver / snapshot 逐步靠拢同一提取结果

### 模板识别

- task 编辑优先使用 task 模板
- override 模板通过模板 ID / 模板来源优先反查
- 避免任务被闪念/block 模板抢走

### 时间计算

- 默认值注入后触发时间联动
- 支持中文字段和常见英文别名
- 加入中文调试日志

### 主题三分法

- `themePath`: 完整主题路径
- `rootTheme`: 根主题
- `leafTheme`: 叶主题
- 视图字段、筛选、缓存支持三分法字段

### 写回与迁移

- 任务写回保留 tag / emoji / 元数据 / checkbox
- 路径变化时支持安全迁移：先写新位置，再删旧记录
- 删除旧记录失败时返回 partial_success

### 保存位置预览

- 编辑中实时计算 OutputPlan
- 提交时携带预览计划
- usecase 保存前重新计算实际计划
- 预览与实际不一致时取消保存

## 不再建议继续扩展的部分

当前不建议继续大改：

- 自动复杂合并原任务未知 token
- 自动跨库迁移
- 大规模删除 legacy 代码
- 将所有 block 类型一次性切到 snapshot 主链

这些可以等一轮完整手测后再推进。
